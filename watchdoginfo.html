How Tests Are Performed

Watchdog evaluates resource usage with precise tests, benchmarked against global standards:

  • CPU Efficiency: Executes 500K iterations of random calculations, scoring execution time (4-8). Similar to Green Software Foundation’s Carbon-Aware SDK, but client-side, reducing server load.
  • Memory Usage: Allocates a 200K-element array, measuring heap change (<2MB best). Contrasts with Persefoni’s data-heavy Scope 3 analytics, prioritizing lightweight operation.
  • Power Usage: Estimates wattage (40W baseline), like Microsoft’s Emissions Impact Dashboard, but runs locally for instant results.
  • Battery Impact: Scores battery level (>50% best) via Battery API, impacting carbon scores, akin to Google’s energy-efficient routing in Maps.

Tests produce an Eco Score (0-100) and Carbon Footprint (kg CO2/h), calibrated for mobile/desktop. Unlike Sphera’s centralized reporting, Watchdog’s real-time, client-side metrics empower developers instantly.[](https://medium.com/energy-web-insights/energy-web-watttime-and-green-software-foundation-unveil-open-source-carbon-aware-nomination-to-6e64bcdc5a40)[](https://sustainability.google/)[](https://plana.earth/academy/best-carbon-accounting-software-2023)
